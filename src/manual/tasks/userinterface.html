UserInterface

Description

Creates simple user interfaces to display a message or request an input from the user.

User interfaces can include labels, text fields, file/directory selections and check boxes. The input elements are defined by the nested elements of this task. Each input element defines an associated property that is set to the element value when the user interface is validated.

The user will be able to continue the build only when inputs of all elements are considered as valid. The user can otherwise choose to abort the build.

A file attribute can be set to load the properties from / save the properties to before / after the user interface is displayed. This enables to automatically retrieve values entered by the user in subsequent builds.

Parameters

Attribute Description Required
title The title of the window. No
file The properties file where to read/save the properties. No

Parameters specified as nested elements

Input elements

Nested elements allow several (input) interfaces to be specified. The order of the nested elements defines the order of the graphical elements (from top to bottom) in the user interface.

  • <label> element

Displays some text with an optional icon on the left.
Advanced styling can be obtained by using HTML tags. The text must then start and end with an <html> tag.

Parameters
Attribute Description Required
text The text to display. Yes,
unless text is included in a character section within this element.
icon The icon to display. This can have one of four values ("question", "information", "warning" or "error"). No
  • <checkbox> element

Displays a check box.

Parameters
Attribute Description Required
text The text of the check box. Yes,
unless text is included in a character section within this element.
property The name of the property that will be set to the value of the check box. Yes
default A default value to initialize the check box to (only if the property wasn't loaded from the properties file). No; defaults to false.
  • <buttongroup> element

Defines a button group. The button group element manages the selected/unselected state for a set of radio buttons. For the group, it guarantees that only one button can be selected at a time.
A button group is relevant only with <radiobutton> children.

Parameters
Attribute Description Required
property The name of the property that will be set to the index of the selected radio button.
The selection index starts from 0 and identifies children radio button from top to bottom. 		Yes
default A default value to initialize the selection to (only if the property wasn't loaded from the properties file). No; defaults to 0 (first radio button selected).
  • <radiobutton> element

Displays a radio button. Radio buttons can be displayed on their own or managed inside a <buttongroup> to ensure only one button is selected at a time.

Parameters
Attribute Description Required
text The text of the radio button. Yes,
unless text is included in a character section within this element.
property The name of the property that will be set to the value of the radio button. Only required if not part of a button group.
default A default value to initialize the radio button to (only if the property wasn't loaded from the properties file). No; defaults to false and ignored if part of a button group.
  • <textfield> element

Displays a text field with a label on the left.

Parameters
Attribute Description Required
text The text of the label. Yes
property The name of the property that will be set to the value of the text field. Yes
allowempty Allow empty value.
Note: If the value is empty, the property won't be set.		 No; defaults to false.
default A default value to initialize the text field to (only if the property wasn't loaded from the properties file). No
tooltip The text to display when the cursor lingers over the component. No
icon The icon of the label. This can have one of four values ("question", "information", "warning" or "error"). No
  • <passwordfield> element

Displays a password field with a label on the left.

Parameters
Attribute Description Required
text The text of the label. Yes
property The name of the property that will be set to the value of the password. Yes
allowempty Allow empty value.
Note: If the value is empty, the property won't be set.		 No; defaults to false.
tooltip The text to display when the cursor lingers over the component. No
icon The icon of the label. This can have one of four values ("question", "information", "warning" or "error"). No
  • <fileselection> element

Displays a file (or directory) selection interface.

Parameters
Attribute Description Required
text The text of the label. Yes
property The name of the property that will be set to the value of the text field. Yes
mode The selection mode. This must whether be "file" or "directory". No; defaults to "directory".
allowempty Allow empty value.
Note: If the value is empty, the property won't be set.		 No; defaults to false.
allownotexist Allow to enter file / directory path that do not exists. No; defaults to false.
default A default value to initialize the selection to (only if the property wasn't loaded from the properties file). No
tooltip The text to display when the cursor lingers over the component. No
icon The icon of the label. This can have one of four values ("question", "information", "warning" or "error"). No
